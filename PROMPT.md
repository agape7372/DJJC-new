# 게임 고품질 구현 프롬프트

> 이 프롬프트는 게임 기능 구현 시 "손맛"과 품질을 보장하기 위한 가이드입니다.

---

## 사용법

새로운 기능을 요청할 때 이 프롬프트를 복사하여 `[요청 사항]` 부분에 원하는 기능을 작성하세요.

```
"Ultrathink"
```

---

## 프롬프트

```markdown
# Role: Creative Tech Lead (Game Feel Expert)

[Task]
현재 프로젝트의 코드 스타일과 아키텍처를 완벽히 유지하면서, 아래 [요청 사항]을 구현해 주세요. 단순한 기능 작동을 넘어, 플레이어가 느낄 "재미"와 "시각/청각적 피드백(Juiciness)"을 최우선으로 고려해야 합니다.

[Context & Constraints]
- 장르: 2D 도트 타이쿤 (Web/Mobile)
- 핵심 가치: 쾌적한 조작감, 즉각적인 UI 피드백, 귀여운 도트 감성
- 주의: 기존 로직(특히 데이터 저장, 재화 계산)을 파괴하지 않는 안전한 코드를 작성하십시오.

[구현 가이드라인]
1. **Game Feel (손맛)**: 기능을 구현할 때 Tweening, 파티클, 화면 흔들림, 사운드 트리거 등 사용자의 행동에 즉각 반응하는 'User Feedback' 요소를 적극적으로 코드에 포함하십시오.
2. **Clean Code**: 기존 코드의 변수 명명법과 구조를 따르고, 스파게티 코드가 되지 않도록 모듈화하십시오.
3. **Robustness**: 예외 상황(입력 중복, 값의 오버플로우 등)에 대한 방어 코드를 작성하십시오.
4. **CLAUDE.md, progress.md**를 참고하십시오.

[Output Format]
1. **Vibe Strategy**: 어떻게 "손맛"을 살렸는지에 대한 짧은 구현 의도 설명.
2. **Production Code**: 복사-붙여넣기로 바로 동작하는 전체 코드 블록.

[요청 사항]
(여기에 원하는 기능을 구체적으로 적으세요)
```

## 요청 사항 예시

### 예시 1: 시각 효과
```
손님이 돈을 낼 때 동전이 촤르륵 쏟아지는 효과를 넣어줘
```

### 예시 2: 시스템 구현
```
날짜/시간 시스템을 추가하고 싶은데, 게임 내 활동을 하면 에너지가 닳고
"운영 n일째"로 넘어가는 방식으로 구현해줘. 아이디어 3가지 제시 후 선택하게 해줘.
```

### 예시 3: UI 개선
```
업그레이드 구매 시 버튼이 눌리는 애니메이션과
구매 완료 시 축하 파티클이 터지도록 해줘
```

### 예시 4: 밸런스 조정
```
현재 손님 등장 확률이 너무 낮아.
시간대별로 손님 타입 가중치를 조절하고,
피크 타임에는 손님이 더 많이 오도록 해줘.
```

## Game Feel 체크리스트

구현 완료 후 아래 항목을 점검하세요:

### 사운드 (Sound)
- [ ] 버튼 클릭 시 클릭음
- [ ] 성공/실패 시 효과음
- [ ] 중요 이벤트 시 알림음
- [ ] 액션 수행 시 피드백 사운드

### 파티클 (Particle)
- [ ] 성공 시 축하 이펙트
- [ ] 에러 시 경고 이펙트
- [ ] 상태 변화 시 전환 이펙트
- [ ] 보상 획득 시 빛나는 이펙트

### 애니메이션 (Animation)
- [ ] 버튼 누름 효과 (scale down)
- [ ] 화면 전환 페이드/슬라이드
- [ ] 숫자 변화 시 카운트 업/다운
- [ ] 중요 요소 펄스/글로우 효과

### 촉각 피드백 (Haptic)
- [ ] 성공 시 화면 흔들림 (긍정적)
- [ ] 실패 시 화면 흔들림 (경고)
- [ ] 큰 보상 시 강한 흔들림

### 시각 피드백 (Visual)
- [ ] 비활성 버튼 회색 처리
- [ ] 호버/포커스 상태 표시
- [ ] 진행률 바/게이지 애니메이션
- [ ] 토스트/알림 메시지

## 품질 기준

### 필수 (Must Have)
- 기존 코드 스타일 준수
- 에러 핸들링 포함
- 사운드 피드백 1개 이상
- 파티클 피드백 1개 이상

### 권장 (Nice to Have)
- 화면 흔들림 효과
- 트윈 애니메이션
- 로딩/진행 상태 표시
- 토스트 메시지

### 금지 (Don't)
- console.log 남기기 (디버그용 제외)
- 하드코딩된 매직 넘버
- 기존 저장 데이터 호환성 파괴
- Optional chaining(?.) 남용
